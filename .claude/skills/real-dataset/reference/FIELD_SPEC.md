# 신청서 데이터셋 — 필드 스펙 & 케이스 레퍼런스

Claude/사람이 신청서 xlsx 를 만들거나 자동입력 동작을 이해할 때 참고하는 문서.
데이터 흐름: **이미지/입력 → 스펙(dict) → `build_real.py` → 세로형 xlsx → `parser.py`(매핑) → `app.py`(라이브 폼 자동입력·검증)**

GUI 로 만들려면 `python make_gui.py` (또는 `run_maker.bat`). 스크립트로 만들려면
`python .claude/skills/real-dataset/scripts/build_real.py spec.json --outdir real_MMDD`.

---

## 1. 스펙 필드 (build_real 입력 키)

모두 문자열. 없으면 생략 가능. GUI 필드와 1:1 대응.

| 키 | 뜻 | 예 / 값규칙 |
|---|---|---|
| `gubun` | 전기/수소 | `전기`·`수소` (생략 시 차종으로 추정: 넥쏘→수소) |
| `contract_day` | 계약일자 | `YYYY-MM-DD`. **폴더명 real_MMDD 를 이 값에서 유추** |

| `name` | 성명/기관명 | 파일명 `신청서_<name>.xlsx` |
| `birth` | 생년월일 | `YYYY-MM-DD` (**전체날짜 유지**) |
| `sex` | 성별 | `남`·`여` |
| `busi_no` `busi_nm` | 사업자번호·명 | 개인사업자/단체일 때 |
| `model` | 신청차종 | **사이트 정식명에 가깝게** (§4 참고) |
| `cnt` | 신청대수 | 숫자 |
| `delivery` | 출고예정일자 | `YYYY-MM-DD` |
| `subsidy` | 보조금금액 | 표시용(폼은 자동계산) |

| `phone` | 전화번호 | **보통 `.`** (§5) |
| `mobile` | 신청자 휴대폰 | 실제 번호는 여기에 |
| `email` | 이메일 | 없으면 `.` |

| `first_reg` | 차량 최초등록일 | `YYYY-MM-DD` |
| `own_start` `own_end` | 소유기간 시작·종료 | `YYYY-MM-DD` |
| `fuel` | 유종 | `LPG`·`경유`·`휘발유` |
| `scrap_car_no` | 폐차 차량번호 | 예 `12가3456` |

| `contract_no` | 계약번호 | 예 `AB1234XX0000` |

## 2. 케이스별 필수/특이 항목

| 케이스 | 핵심 값 | 폼에서 열리는 것 |
|---|---|---|
| **개인** | `req_kind=개인` | 기본 |
| **개인사업자/단체** | `req_kind=개인사업자`+`busi_no`/`busi_nm` | 사업자번호·기관명 칸 노출(req_kind=B/G) |
| **택시(개인택시)** | `taxi_yn=Y` | 첨부 A7 포함(§6) |
| **생애최초** | `first_buy_yn=Y` | 라디오만 |
| **사회계층** | `social_yn=Y`+`social_kind` | `social_kind` 드롭다운 노출 |
| **전환지원금** | `exchange_yn=Y`+폐차정보 6칸 | §3 폐차 정보 행 + 첨부 7곳(§6) |
| **수소(FCEV)** | `gubun=수소` (넥쏘 등) | URL `/h2/`, 일부 옵션 다름 |

## 3. 전환지원금 폐차 정보 (라이브 폼 동작)

`exchange_3year_yn=Y` → **폐차대수(`exchange_3year_cnt`) 입력** →
[확인] = `create3yearNewCarInfo()` 가 '전환지원금 폐차 정보' 행을 생성.
생성된 행 입력칸은 **id 없이 `name` 기반**(같은 name 의 숨은 템플릿 존재 → 보이는 요소만 채움):

| 화면 라벨 | name | 스펙 키 |
|---|---|---|
| 직전 소유주 | `bf_owner_nm` | prev_owner |
| 차량 최초 등록일 | `exchg_delivery_day` | first_reg |
| 소유기간시작일 | `own_start_dt` | own_start |
| 소유기간종료일 | `own_end_dt` | own_end |
| 유종 | `fuel` (select) | fuel |
| 차량번호 | `exchg_vh_num` | scrap_car_no |

유종 코드: **LP=LPG · DS=경유 · GS=휘발유** (parser `FUEL_MAP`).

## 4. 신청차종 매칭 팁

사이트 옵션은 전부 **"더 뉴 아이오닉5 …"**. 같은 19인치 롱레인지가 2종이라
이미지 문구만으론 애매 → 정식명으로 넣어 유일 매칭시킨다.

| 데이터에 넣을 이름 | 매칭 코드 |
|---|---|
| `더 뉴 아이오닉5 2WD 롱레인지 19인치/일반승용` | THENEW5_19_2 (기본, 빌트인캠 적용) |
| `더 뉴 아이오닉5 2WD 롱레인지 19인치 빌트인 캠 미적용/일반승용` | THENEW5_19_3 |

정확 텍스트가 없으면 app.py 가 **가장 비슷한 것 + 🔴빨간 경고**로 넣으니 사람이 확인.

## 5. 연락처 규칙

- 종이 신청서의 **전화번호가 실제로는 휴대폰 번호**인 경우가 많다.
  → 폼의 `전화번호(phone)` = **`.`**, `휴대폰(mobile)` = 실제 번호.
- 이메일 없으면 `.`(필수칸 표시). 값 `.` 은 그대로 입력한다.
- 생년월일은 **전체날짜(YYYY-MM-DD)** 로.

## 6. 첨부파일 규칙 (임시저장 후 화면)

`app.py` 가 화면에 **보이는 첨부칸을 자동 감지**해 하나의 PDF 를 올린다.
`A7`(우선순위 증빙자료)은 **우선순위 대상만** 포함.

| 케이스 | 판별 | 첨부칸 |
|---|---|---|
| 일반 | — | A·A2·A3 (3곳, A7 제외) |
| 택시 | `taxi_yn=Y` | A·A2·A3·**A7** (4곳) |
| 전환지원금 | 전용첨부 A17/A18/A19 존재 | A·A2·A3·A7·A17·A18·A19 (7곳) |

첨부칸 뜻: A=신청서+개인정보동의서 · A2=구매계약서 · A3=등본/등기부 ·
A7=우선순위 증빙 · A17=말소사실증명서 · A18=자동차 등록원부 · A19=가족관계증명서.

## 7. 자동화 안 하는 것 (사람 몫)

- **암호 역순 입력**(캡차성) · **저장/임시저장 최종 확인** · **제출**
- 차종이 '가장 비슷한 것'으로 들어간 🔴 경고 항목은 반드시 사람이 확인.
