# real-dataset — 신청서 이미지 → 전기/수소 데이터셋

보조금 신청서 이미지를 받아, 실전 양식(`examples/real_sample_ev.xlsx`)과 동일한 2D 그리드
엑셀로 만들어 `real_<MMDD>/신청서_<이름>.xlsx` 에 저장한다. 최상단 `구분` 셀에 전기/수소를
넣어 `parser.py` 가 `car_kind` 를 자동 인식하게 하는 것이 핵심.

## 워크플로우

1. **이미지에서 값 추출.** 첨부된 신청서 사진의 모든 칸을 읽는다. 표의 라벨-값을 그대로 옮기되,
   흐리거나 자신 없는 값은 추측하지 말고 사용자에게 확인한다.

2. **전기/수소 분류.** 신청차종으로 판단해 `gubun` 을 명시적으로 정한다.
   - `넥쏘`/`디올넥쏘`/`수소`/`FCEV` → **수소**
   - 아이오닉·캐스퍼·EV6·코나 일렉트릭·스타리아/캐스퍼 일렉트릭 등 → **전기**
   - 이미지에 `구분` 칸이 따로 있으면 그 값을 우선한다.
   - 애매하면 사용자에게 묻는다. (스크립트도 넥쏘→수소, 그 외→전기로 자동 추정하지만
     되도록 스펙에 직접 넣는다.)

3. **스펙 JSON 작성.** 스크래치패드에 `spec.json` 을 쓴다. 한 명이면 객체, 여러 명이면 배열.
   전체 필드·케이스·사이트 매핑·전환지원금·첨부 규칙은 [reference/FIELD_SPEC.md](reference/FIELD_SPEC.md),
   빈 템플릿은 [reference/spec_template.json](reference/spec_template.json) 참고. 값 규칙:
   - 날짜는 `YYYY-MM-DD`. 폴더명 `real_MMDD` 는 `contract_day`(계약일자)에서 자동 유추.
   - 이미지에서 `.` 으로 비워둔 칸(전화/이메일 등)은 `.` 그대로 넣는다(필수칸 표시).
   - 개인사업자면 `req_kind:"개인사업자"` + `busi_no`/`busi_nm`.
   - 신청자 휴대폰은 `mobile`, 담당자 휴대폰은 `contact_mobile` 로 구분(헷갈리기 쉬움).
   - 이미지 하단 암호/역순은 캡차성 단계라 자동입력 대상이 아니지만, 기록용으로 `pw`/`pw_rev` 에 넣어도 된다.

4. **생성.** 프로젝트 루트에서 실행(경로에 한글이 있으면 따옴표로 감싼다):
   ```bash
   python .claude/skills/real-dataset/scripts/build_real.py <스크래치패드>/spec.json
   ```
   - 2건 이상이면 `신청서_통합_N건.xlsx`(신청서목록, 코드값 P/B/G·M/F·전기/수소)도 자동 생성.
   - 폴더를 강제 지정하려면 `--outdir real_0707`.

5. **검증.** 만든 파일을 파서로 돌려 매핑을 확인하고 사용자에게 요약 표로 보고한다:
   ```bash
   python parser.py "real_<MMDD>/신청서_<이름>.xlsx"
   ```
   특히 `car_kind`(전기/수소)와 `req_kind`(P/B/G)가 의도대로 잡혔는지 확인한다.
   (Windows 터미널은 한글이 깨져 보일 수 있으나 파일 값 자체는 정상이다.)

## 주의
- 이미지 값은 **그대로** 옮기는 게 원칙. 임의 보정/정규화 금지(날짜 포맷 통일만 허용).
- 자신 없는 판독값은 만들어 넣지 말고 사용자에게 확인 후 진행한다.
- `구분`(전기/수소)은 실제 폼이 어느 쪽으로 열려 있는지와 반드시 일치해야 한다 — 사고 방지 필드.
