# CarMacro

전기차·수소차 **보조금 신청서(ev.or.kr)** 자동입력 도구.
세로형 엑셀(한 파일 = 한 신청서)을 읽어, 이미 로그인·진입해 둔 신청서 폼에
자동으로 채워 넣고 **입력값을 다시 읽어 대조(검증)** 한다. 저장/제출은 하지 않는다(사람 몫).

> ⚠️ 실제 보조금 신청에 쓰는 도구입니다. 잘못 넣으면 큰일이라, **빠른 입력보다 "애매한 건 빨갛게 경고"** 에 무게를 둡니다. 빨간 로그(❌)는 반드시 사람이 확인하세요.

---

## 무엇을 자동으로 하나
- 세로형 엑셀의 한글 라벨을 읽어 사이트 필드로 매핑 (계약일·신청유형·성명·생년월일·성별·사업자번호/명·신청대수·출고예정일·휴대폰·담당자·계약번호 등)
- **차종·사회계층 유형**: 페이지의 실제 드롭다운과 **한글 텍스트로 대조** (정확히 없으면 가장 비슷한 것 + 🔴빨간 경고)
- **주소**: 도로명으로 주소검색 팝업을 자동 실행 → 결과가 정확히 1개면 우편번호+주소 자동 입력
- 입력 후 **되읽기 검증**: 폼 값을 다시 읽어 엑셀과 한 칸씩 대조, 다르면 ❌
- **전기/수소 구분 가드**: 엑셀 `구분`(전기/수소)과 열린 폼(URL `/h2/` = 수소)이 어긋나면 입력 거부
- 자동 못 하는 것(우편번호 결과 다수, 교체차량, 암호 역순 등)은 **handoff 목록**으로 출력

## 사람이 해야 하는 것
1. 크롬을 **디버그 포트 9222** 로 띄우고 ev.or.kr 로그인 → **지자체 선택 → 신청서 작성 폼** 진입
2. 🔴 빨간 로그(차종이 "가장 비슷한 것"으로 들어갔는지 등) 확인
3. handoff 항목 직접 입력: 교체(폐차)차량, 경유화물차 폐차 미이행여부, 암호 역순, 첨부서류
4. 최종 검토 후 **저장/제출** (도구는 제출하지 않음)

## 설치 (다른 노트북에서 처음 세팅)
사전 준비: **Python 3.11+**, **Chrome** 설치.

```bat
git clone https://github.com/Gachon-junghyun/CarMacro.git
cd CarMacro
python -m pip install -r requirements.txt
```
> selenium 4.6+ 는 크롬드라이버를 자동으로 받아줍니다(Selenium Manager). 별도 설치 불필요.

## 실행
1. **크롬을 디버그 포트로 띄우기** (이미 떠 있는 크롬은 모두 종료 후):
   ```bat
   run_chrome.bat
   ```
   또는 직접:
   ```bat
   "C:\Program Files\Google\Chrome\Application\chrome.exe" --remote-debugging-port=9222 --user-data-dir="%USERPROFILE%\chrome-debug-carmacro"
   ```
   이 크롬에서 **ev.or.kr 로그인 → 신청서 작성 폼**까지 들어갑니다.
2. **앱 실행**:
   ```bat
   python app.py
   ```
3. GUI에서 **[엑셀 불러오기]** → 신청서 엑셀 선택 → **[시작]**
   - 폼이 떠 있어야 자동 입력됩니다. 빈 폼이 뜨면 자동으로 채웁니다(또는 [▶ 다음 건 입력]).
   - 입력 중에는 **마우스·키보드를 건드리지 마세요**(주소 팝업 포커스 튐 방지).

## 예시 데이터 만들기
```bat
python examples\make_dummies.py
```
`examples\real_sample_ev.xlsx`(전기, 캐스퍼), `examples\real_sample_h2.xlsx`(수소, 디올뉴넥쏘) 생성.
파서만 단독 확인: `python parser.py examples\real_sample_ev.xlsx`

## 엑셀 양식 (세로형)
한 파일 = 한 신청서. 라벨 셀 옆(오른쪽)에 값을 둔다. 주요 라벨:
`구분(전기/수소)`, `계약일자`, `신청유형(개인/개인사업자/단체)`, `성명`, `생년월일`, `성별`,
`사업자번호`, `사업자명`, `신청차종`, `신청대수`, `출고예정일자`, `주소`, `이하주소`,
`전화번호`/`이메일`/`휴대폰`, `사회계층여부`+`유형`, `우선순위 배정.집행 선택`,
`담당자성명`, `휴대폰(담당자)`, `계약번호`.
> 차종명은 **사이트 정식명에 가깝게**. 정식 목록은 `reference/models_reference.txt` 참고.

## 폴더 구조
```
app.py                  메인 GUI + 자동입력 + 검증
parser.py               세로형 엑셀 파서
test_validate.py        검증 로직 단위 테스트
reference/              차종 목록 등 참고자료
examples/               더미 생성 스크립트 + 예시 엑셀
tools/                  사이트 구조 진단용 개발 스크립트(이 PC/세션 한정)
```

## 주의/한계
- ev.or.kr 의 화면 구조(필드 id, 팝업 함수)에 의존합니다. 사이트가 바뀌면 `tools/inspect_*.py` 로 다시 따야 합니다.
- 전기/수소 폼은 일부 항목이 다릅니다(예: 수소엔 개인사업자·우선순위 옵션이 없거나 마감으로 비활성). 그런 값은 ❌ 빨간 경고로 알리고 사람이 처리합니다.
- **저장·제출·암호(캡차) 입력은 자동화하지 않습니다.**
